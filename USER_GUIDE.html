Maqueen Lab

A web-based BLE component lab for the DFRobot Maqueen Lite v4 — every actuator and sensor, live in your browser.

▶ Open the app 🤖 Meet your robot ⭐ GitHub

🚀 Get going in 3 minutes

Flash the firmware, pair over Bluetooth, hit a button.

1

Flash the firmware

Open makecode.microbit.org, add the pxt-maqueen extension, paste the code from the app's 🤖 Firmware button, download the .hex to your micro:bit.

2

Connect

Power the robot. Open Maqueen Lab in Chrome / Edge. Click Connect and pick BBC micro:bit [xxxxx].

3

Play

Land on the 🤖 Maqueen tab. Tap a direction, drag a slider, pick a color. Watch the live sensor strip up top react.

✨ What you can do

Six panels in the Maqueen tab, each driving a real piece of the robot.

🛞

Drive the wheels

Direction pad + speed slider. Drag while moving — speed updates instantly.

🦾

Pose the servos

Two sliders, three presets. Pick a Mechanic Kit and the panel reshapes itself.

💡

Toggle the LEDs

Two simple LEDs (left/right), toggle, blink-5×, all off.

🌈

Paint with NeoPixels

Four addressable RGB pixels. Per-pixel colors or a rolling rainbow animation.

🔊

Play music

Piano-key notes (C–B), or any frequency + duration you type.

👀

Read every sensor

Line eyes, ultrasonic distance, IR remote, accelerometer — live on the strip up top.

📡

Every command is echo-confirmed

Each button press sends a sequence-numbered verb (#42 SRV:1,90). The micro:bit replies ECHO:42 SRV:1,90. The bench display in the live strip shows sent / echoed / lost / average round-trip — so you always know if BLE dropped a packet.

❓ When things go sideways

Click any card to expand.

🛑 "Cannot add neopixel extension" in MakeCode

MakeCode blocks the standard neopixel extension when bluetooth is already added (timing conflict on micro:bit). Three workarounds:

  • Skip RGB for now — comment out the NeoPixel block in the firmware. Everything else (motors, servos, sensors, LEDs, buzzer) works perfectly. RGB commands become serial-log no-ops.
  • Try a BLE-safe fork: paste https://github.com/Bohwaz/pxt-neopixel in Extensions → search.
  • Share your working sketch: if NeoPixels worked for you in another project, send the neopixel.create(...) line so the firmware can match it.
📡 "echo timeout" warnings in the BLE Console

Every command times out after 500 ms if no ECHO:N reply arrives. Common causes:

  • Robot not connected (check the green Connected chip)
  • Battery low — symptom: BLE drops mid-session
  • Laptop too far from the robot for the first pair (move within 1 m)
  • Firmware version mismatch — re-flash from the 🤖 Firmware modal
🚫 "UNKNOWN_VERB" reply on a command you expected to work

The firmware doesn't recognize the verb. You're probably running an older firmware than the app expects. Open the 🤖 Firmware modal in the Connect card → Copy Code → paste into MakeCode → flash.

💤 Page looks broken or stuck on an old version

The service worker cached old assets. Two fixes:

  • Click 🗑️ Reset in the header (clears localStorage + service worker)
  • Or hard-reload with Ctrl+Shift+R
📜 Show me every BLE verb

Full reference (commands + replies + bridged bit-playground verbs):

M:L,R           motors signed -255..255 each
STOP            brake
LED:i,s         simple LED i=0|1, s=0|1
RGB:i,r,g,b     NeoPixel i=0..3 or *
SRV:i,a         servo i=1|2, a=0..180
BUZZ:f,ms       tone (f=0 silences)
LINE?           reply LINE:l,r
DIST?           reply DIST:cm
IR?             reply IR:code
LOG:n           USB serial verbosity 0..3
BENCH:PING      reply BENCH:PONG
HELLO           reply HELLO:<ver>
TEXT:msg        scrolling text on LED matrix
CMD:icon        HEART/SMILE/SAD/FIRE/UP/DOWN/LEFT/RIGHT/CLEAR
SERVO1:angle    alias of SRV:1,angle
SERVO2:angle    alias of SRV:2,angle

Streams from the firmware: ACC:x,y,z, TEMP:°C, LIGHT:n, SOUND:n, BTN:A:1, BTN:B:1.

📌 Which pin drives what?

Quick cheat sheet — see pinout.html for the full evidence trail.

P0    Buzzer (and Gravity port)
P1    Ultrasonic TRIG (and Gravity port)
P2    Ultrasonic ECHO (and Gravity port)
P8    Simple LED Left
P12   Simple LED Right
P13   Line sensor Left
P14   Line sensor Right
P15   NeoPixel data (4 pixels)
P19   I2C SCL — motor driver, servos
P20   I2C SDA — same bus
🛠️ I want to extend it (developer notes)

File layout:

index.html             main app
firmware/v1-...ts      Maqueen firmware
js/ble.js              UART layer — DO NOT EDIT
js/ble-scheduler.js    seq + echo wrapper
js/maqueen-panel.js    live sensor strip
js/maqueen-tab.js      Maqueen tab controls
tools/bump-firmware.py auto-bump version + re-embed

Add a new firmware verb:

  1. Add a branch in the dispatch if/else in firmware/v1-maqueen-lib.ts.
  2. Run python tools/bump-firmware.py (bumps patch version + re-embeds the blob in index.html).
  3. Add UI in js/maqueen-tab.js, send via window.bleScheduler.send('VERB:args').
  4. If it's a query, parse the reply in js/maqueen-panel.js's onReply.

Architecture rule: never modify js/ble.js. Wrap, never replace.

1. الرؤية

ليست « جهاز تحكّم بالروبوت » — بل مختبر مكوّنات. كلّ محرّك وحسّاس على Maqueen Lite له صفحة Explorer مخصّصة حيث يستطيع الطفل:

  • رؤية المكوّن مُمثَّلاً بصرياً بشكل واقعيّ.
  • التحكّم به عبر منزلقات، أزرار، منتقيات ألوان.
  • قراءة الكود الذي يقوده مباشرةً (TypeScript مع مكتبة maqueen.* أوّلاً، ثمّ pins.* الخامة).
  • تنفيذ تحدّيات صغيرة موجَّهة (« اجعل الذراع تلوّح »، « جِد المركز »).
  • مشاهدة بيانات الحسّاسات تتفاعل مع الأوامر على رسم بيانيّ مشترك للقياس عن بُعد.

كلّ التحكّم يمرّ عبر UART BLE. كلّ أمر مؤكَّد بـ echo.

3. قرارات معماريّة

3.1 بروتوكول BLE — تسلسل + تأكيد بـ echo

كلّ أمر يحمل رقم تسلسل. micro:bit يُعيده echo. تطبيق الويب يتحقّق من الذهاب والإياب.

Web → bit:   #42 SRV:1,90
bit → Web:   ECHO:42 SRV:1,90
bit → Web:   DONE:42                (اختياريّ، للعمليّات البطيئة)

جانب الويب (js/ble-scheduler.js):

  • pendingCmds: Map<seq, {verb, ts, resolve, reject}>
  • Timeout 500 ms ← reject + وميض أحمر UI + log console
  • لوحة benchmark: مُرسَل / تمّ echo / مفقود / متوسّط الكُمون
  • تيّارات الحركة: coalesce قبل تخصيص seq

UI لكلّ أمر (في لوحة الكود):

ble.send('SRV:1,90')   ✅ 47ms
ble.send('SRV:1,95')   ⏳
ble.send('SRV:1,100')  ❌ timeout

3.2 logging مزدوج عبر USB المسلسل

الـ firmware يعكس كلّ شيء إلى console USB المسلسل بسرعة 115200 baud:

[boot] MaqueenApp firmware v0.1.0 built 2026-04-26 01:23 UTC
[boot] hardware: Maqueen Lite v4
[boot] BLE name: BBC micro:bit [vagiv]
[ble]  connected
[rx]   #42 SRV:1,90
[exec] maqueen.servoRun(S1, 90)
[tx]   ECHO:42 SRV:1,90

مستوى التفصيل عبر BLE: LOG:0 (صامت)، LOG:1 (rx/tx)، LOG:2 (+exec)، LOG:3 (+sondage حسّاسات).

3.3 بانر البناء

BUILD_VERSION و BUILD_DATE سطرا const في رأس firmware. حرّر يدوياً قبل كلّ flash. البانر يُطبَع عند الإقلاع.

3.4 حركات يقودها الويب

firmware micro:bit بسيط: لا حلقات على اللوحة. كلّ الحركات (sweep, blink, rainbow) يولّدها تطبيق الويب بإرسال أوامر متدفّقة.

سقف معدّل BLE: ~8 cmd/ثانية بشكل موثوق. الـ scheduler:

  • يُكَتِّل drag السريع للمنزلق إلى آخر قيمة.
  • يُحدّ المعدّل لكلّ خاصّيّة.
  • سجلّ الحركات: scheduler.animate(verb, frame → value, fps) متزامن مع معدّل BLE، ليس 60 Hz.

المرئي يبقى عند 60 fps محلّياً — الطفل لا يشعر أبداً بـ lag، حتّى عندما يُؤكِّد BLE أبطأ.

3.5 مساران للـ firmware (تقدّم تعليميّ)

firmware/
├── v1-maqueen-lib.ts        ← يستخدم maqueen.*  (افتراضي، شُحن v0.1.0)
└── v2-raw-pins.ts           ← يستخدم pins.*    (متقدّم، يُشحَن لاحقاً)

نفس الأفعال UART في كليهما ← نفس تطبيق الويب يعمل مع أيّ منهما. لوحة الكود تُظهر الترجمتَين جنباً إلى جنب لكلّ فعل.

4. نمط Component Explorer

كلّ مكوّن يستخدم نفس التخطيط:

┌─────────────────────────────────────┐
│  Visual (live, animated, 60fps)     │  ~40%
├─────────────────────────────────────┤
│  Controls (sliders, buttons)        │  ~30%
├─────────────────────────────────────┤
│  Live code (TS using maqueen lib) 📋│  ~20%

اللقاء الأوّل: عرض تلقائيّ لمدّة ثانيتَين عند فتح كلّ Explorer لأوّل مرّة.

لوحة الكود تُظهر نسختَين جنباً إلى جنب: maqueen.* (lib) و pins.* (خام). الطفل يرى التجريد، ثمّ الآليّة.

« أبرِز ما تغيّر »: عندما يتحرّك المنزلق، السطر الذي تغيّر يومض.

5. مواصفات Explorer لكلّ مكوّن

Servo (المرشد — يُبنى أوّلاً)

  • المرئي: SVG من فوق لـ servo نمط Tower-Pro. الذراع تدور حياً مع المنزلق. ذراع شبحيّة تشير للزاوية المستهدفة. مسطرة زاويّة 0–180°. drag دبابيس حدود المعايرة على المسطرة. وضع Sweep = أثر motion blur. صوت طقطقة تروس خفيف.
  • التحكّم: زاوية 0–180، presets (0/45/90/135/180)، وضع Sweep + سرعة، حركة سلسة steps/ms.
  • المعايرة: إيجاد المركز، تحديد min/max المستخدم، حفظ لكلّ منفذ (S1/S2) في localStorage، مفهرسة بمعرّف جهاز BLE.
  • الفعل: SRV:i,a (i ∈ 1,2 ؛ a ∈ 0..180).

LED بسيط

  • المرئي: قبّة LED 5mm واقعيّة. OFF = زجاج خافت. ON = صعود حراريّ + هالة bloom + جسيمات فوتونيّة.
  • التحكّم: toggle ON/OFF، blink Hz، نبض، نمط fade.
  • vignette: لوحة Maqueen صغيرة تُظهر أيّ LED مضاء.
  • الفعل: LED:i,s (i ∈ 0,1 ؛ s ∈ 0,1).

RGB (4 px)

  • المرئي: عرض من فوق للوحة Maqueen مع 4 NeoPixels في مواقعها الحقيقيّة. نقر أيّ بكسل ← عجلة HSV مخصّصة. السطوع يقود نصف قطر bloom.
  • التحكّم: لون لكلّ بكسل، toggle « كلّها متطابقة »، السطوع، حركات (rainbow / chase / breathe / fade) بسرعة.
  • الفعل: RGB:i,r,g,b (i ∈ 0..3 أو * للكلّ).

المحرّكات

  • المرئي: روبوت من فوق مع SVGs للعجلات. الأشعّة تدور بمعدّل ∝ سرعة. Reverse = الدوران معكوس. خطوط حركة عند السرعات العالية.
  • التحكّم: سرعة −255..+255، اتجاه، فرامل، سرعة المنحدر.
  • المعايرة: trim لكلّ عجلة (offset للسير مستقيماً)، مُحفَظ.
  • الفعل: M:L,R (مُوقَّع −255..255 لكلّ).

صفّارة

  • المرئي: SVG سمّاعة بموجات صوتيّة متمركزة (نصف القطر = حجم، اللون = تردّد). كيبورد بيانو في الأسفل. Web Audio يعزف نفس النغمة في المتصفّح.
  • التحكّم: تردّد، مدّة، نغمات presets، منتقي ألحان.
  • الفعل: BUZZ:f,ms.

أمواج فوق صوتيّة

  • المرئي: قوس مسح سونار من أمام الروبوت. نقطة مسافة مُثَبَّتة على القوس. شبكة قطبيّة (10–100 سم). أثر آخر 50 قراءة كنقاط تتلاشى. حدود عتبة قابلة للتخصيص خضراء/صفراء/حمراء. عرض ضخم.
  • التحكّم: معدّل polling، زرّ ping يدويّ، مناطق العتبة.
  • المعايرة: offset الصفر.
  • الفعل: DIST? ← DIST:cm.

حسّاسات الخط

  • المرئي: روبوت من تحت بـ « عينَين ». تحت: شريط أسود قابل للسحب يحرّكه الطفل تحت العينَين ؛ القراءات الحقيقيّة overlay المحاكاة.
  • التحكّم: معدّل polling، log التحوّلات.
  • الفعل: LINE? ← LINE:l,r.

جهاز التحكّم IR

  • المرئي: SVG واقعيّ لجهاز التحكّم. آخر مفتاح مضغوط يومض + يلمع.
  • التحكّم: وضع raw-code / وضع مفتاح مسمّى. وضع تعلّم: الطفل يضغط مفتاحاً فيزيائياً، يسمّيه، التطبيق يُعيِّن الكود.
  • الفعل: حدث push IR:code (أو poll IR?).

Drive Explorer (مركّب — يستخدم محرّكات + قياس accel)

  • المرئي: joystick + زرّ STOP + canvas روبوت شبحيّ (أثر dead-reckoning).
  • القياس عن بُعد: رسم accel 3 محاور حياً مع علامات أوامر عموديّة على timeline.
  • الفعل: M:L,R + stream auto ACC:x,y,z ~20 Hz.

7. طبقة القياس عن بُعد

في Drive Explorer، اللوحة السفليّة = رسم accel 3 محاور حياً مع علامات أوامر.

Acceleration (g)
  X ━━━━━╱╲━━━━━━╱╲╲━━━━     ← أمام، ثمّ يدور
  Y ━━━━━━━━━━━╱──╲━━━━━     ← جانبيّ في المنعطف
  Z ━━━━━━━━━━━━━━━━━━━━     ← baseline جاذبيّة
        ↑           ↑
   [▶ FWD 200]  [↻ L 150]    ← علامات أمر على timeline

الميزات:

  • Auto-scroll للـ timeline (نافذة 10 ث، drag بإصبعَين للتاريخ).
  • Hover على أيّ نقطة ← tooltip بقيمة g + أقرب أمر.
  • نقر علامة ← يُبرز جزء المنحنى بعدها.
  • Pause / play / snapshot (تصدير PNG) / مقارنة الجولات.

قابلة للإعادة عبر explorers عبر js/telemetry.js:

  • Servo: زاوية مأمورة vs زمن + accel إن كان على ذراع متحرّكة.
  • المحرّكات: أمر + accel + (لاحقاً) encoders العجلات.
  • الصفّارة: شكل موجة ميكروفون فعليّ.
  • أمواج فوق صوتيّة: مسافة vs زمن + علامات ping يدويّ.

8. إطار الإضافات (stub في v0.1.0)

احجز فضاء أسماء الأفعال الآن حتّى لا يكسر v0.2.0+ البروتوكول:

ADDON:LIST                       → ADDON:none|<name>,<port>|...
ADDON:READ:<port>                → ADDON:<port>:<value>
I2C:SCAN                         → I2C:0x10,0x3C,0x76,...
I2C:READ:<addr>:<reg>:<n>        → I2C:<addr>:<reg>:<bytes>
I2C:WRITE:<addr>:<reg>:<bytes>   → ECHO

الـ firmware يَشحن هذه الأفعال stubed (إرجاع none / فارغ). إضافة Gravity/I2C add-ons فعليّة لاحقاً = drop-in ملفّات js/addons/<name>.js، بدون bump للبروتوكول.

9. الهويّة البصريّة

  • السمات: نظام 5 سمات (Carbon · Forest · Steel · Paper · Pearl) — أجواء مختبر مخترع ودود
  • الحالات: أخضر/كهرمانيّ/أحمر مع متغيّرات glow هادئة
  • الطباعة: Righteous (display) · Tajawal (body، عربيّ أيضاً) · JetBrains Mono (كود)
  • الوضع: مظلم أوّلاً (Carbon افتراضيّ) ؛ سمتان فاتحتان (Paper, Pearl)
  • الحركة: فيزياء نابض، أبداً خطّيّة
  • الصوت: off افتراضياً، prompt « 🔊 تفعيل »
  • تفاعلات صغيرة: hover scale 1.02، active 0.98، حلقة focus بـ glow
  • اللمس: Vibration API على snap منزلق الموبايل + ضربات الأزرار

طبقة polish (ابنِ مرّة، أعد الاستخدام في كلّ مكان)

  • SVG لوحة Maqueen مشترك (يُستخدم كأساس في كلّ Explorer).
  • نظام جسيمات (~50 سطر).
  • preset فلتر CSS bloom.
  • util فيزياء نابض.
  • ودجت قياس قابل للتخصيص (قوس / علامات / إبرة).
  • طقم صوتيّ (synth Web Audio، بدون ملفّات صوت).

انضباط الـ stack

  • SVG + CSS فقط — لا محرّك 3D.
  • Web Animations API الأصلي — لا مكتبة حركة.
  • JS vanilla — لا framework، يحافظ على الإحساس الخفيف لـ bit-playground.

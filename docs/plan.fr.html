1. Vision

Pas une « télécommande de robot » — un labo de composants. Chaque actionneur et capteur de la Maqueen Lite a sa propre page Explorer où l'enfant peut :

  • Voir le composant visualisé de manière photoréaliste.
  • Le manipuler via curseurs, boutons, sélecteurs de couleur.
  • Lire en direct le code qui le pilote (TypeScript avec lib maqueen.* d'abord, pins.* brut ensuite).
  • Faire des micro-défis guidés (« Fais saluer le bras », « Trouve le centre »).
  • Voir les données capteurs réagir aux commandes sur un graphe de télémétrie partagé.

Tout le contrôle passe par UART BLE. Chaque commande est confirmée par echo.

3. Décisions d'architecture

3.1 Protocole BLE — séquence + confirmation par echo

Chaque commande porte un numéro de séquence. Le micro:bit le retourne en echo. L'app web valide l'aller-retour.

Web → bit:   #42 SRV:1,90
bit → Web:   ECHO:42 SRV:1,90
bit → Web:   DONE:42                (optionnel, pour ops lentes)

Côté web (js/ble-scheduler.js) :

  • pendingCmds: Map<seq, {verb, ts, resolve, reject}>
  • Timeout 500 ms → reject + flash rouge UI + log console
  • Panneau benchmark : envoyés / echo'd / perdus / latence moyenne
  • Streams d'animation : coalescer avant d'assigner seq

UI par commande (panneau code) :

ble.send('SRV:1,90')   ✅ 47ms
ble.send('SRV:1,95')   ⏳
ble.send('SRV:1,100')  ❌ timeout

3.2 Double logging USB série

Le firmware mirror tout vers la console USB série à 115200 baud :

[boot] MaqueenApp firmware v0.1.0 built 2026-04-26 01:23 UTC
[boot] hardware: Maqueen Lite v4
[boot] BLE name: BBC micro:bit [vagiv]
[ble]  connected
[rx]   #42 SRV:1,90
[exec] maqueen.servoRun(S1, 90)
[tx]   ECHO:42 SRV:1,90

Verbosité par BLE : LOG:0 (silencieux), LOG:1 (rx/tx), LOG:2 (+exec), LOG:3 (+sondages capteurs).

3.3 Bandeau de build

BUILD_VERSION et BUILD_DATE sont deux lignes const en tête du firmware. À éditer à la main avant chaque flash. Le bandeau s'imprime au boot.

3.4 Animations pilotées par le web

Le firmware micro:bit est simple : pas de boucles embarquées. Toutes les animations (sweep, blink, rainbow) sont générées par l'app web qui envoie des commandes en flux.

Plafond de débit BLE : ~8 cmd/s de manière fiable. Le scheduler :

  • Coalesce les drags rapides de slider à la dernière valeur.
  • Limite le débit par caractéristique.
  • Registre d'animations : scheduler.animate(verb, frame → value, fps) synchronisé sur le rythme BLE, pas 60 Hz.

Le visuel reste à 60 fps localement — l'enfant ne sent jamais de lag, même quand BLE confirme plus lentement.

3.5 Deux pistes firmware (progression pédagogique)

firmware/
├── v1-maqueen-lib.ts        ← utilise maqueen.*  (défaut, livré v0.1.0)
└── v2-raw-pins.ts           ← utilise pins.*    (avancé, livré plus tard)

Mêmes verbes UART dans les deux → la même app web fonctionne avec l'un ou l'autre. Le panneau code montre les deux traductions côte à côte par action.

4. Patron du Component Explorer

Chaque composant utilise la même mise en page :

┌─────────────────────────────────────┐
│  Visuel (live, animé, 60fps)        │  ~40%
├─────────────────────────────────────┤
│  Contrôles (curseurs, boutons)      │  ~30%
├─────────────────────────────────────┤
│  Code en direct (TS lib maqueen) 📋 │  ~20%
├─────────────────────────────────────┤
│  Essaie : [Préset A] [B] [C]        │  ~10%
└─────────────────────────────────────┘

Premier contact : auto-démo de 2 secondes la première fois que chaque Explorer s'ouvre.

Le panneau code montre deux versions côte à côte : maqueen.* (lib) et pins.* (brut). L'enfant voit l'abstraction, puis le mécanisme.

« Surligner ce qui a changé » : quand un curseur bouge, la ligne de code qui change clignote.

5. Spécifications par composant Explorer

Servo (pilote — construit en premier)

  • Visuel : SVG vue de dessus d'un servo type Tower-Pro. Le palonnier tourne en direct avec le curseur. Bras fantôme indiquant l'angle cible. Règle angulaire 0–180°. Drag des goupilles de calibration sur la règle. Mode Sweep = traînée de motion blur. Son discret de cliquetis d'engrenage.
  • Contrôles : angle 0–180, presets (0/45/90/135/180), mode Sweep + vitesse, mouvement lissé étapes/ms.
  • Calibration : trouver le centre, fixer min/max utilisateur, sauver par port (S1/S2) dans localStorage, indexé par ID d'appareil BLE.
  • Verbe : SRV:i,a (i ∈ 1,2 ; a ∈ 0..180).

LED simple

  • Visuel : dôme LED 5mm photoréaliste. OFF = verre faiblement éclairé. ON = montée en chauffe + couronne de bloom + particules photoniques.
  • Contrôles : toggle ON/OFF, blink Hz, pulse, motif de fade.
  • Vignette : mini-carte Maqueen montrant quelle LED est allumée.
  • Verbe : LED:i,s (i ∈ 0,1 ; s ∈ 0,1).

RGB (4 px)

  • Visuel : rendu vue de dessus de la carte Maqueen avec 4 NeoPixels aux positions réelles. Cliquer un pixel → roue HSV personnalisée. La luminosité pilote le rayon de bloom.
  • Contrôles : couleur par pixel, toggle « tous identiques », luminosité, animations (rainbow / chase / breathe / fade) avec vitesse.
  • Verbe : RGB:i,r,g,b (i ∈ 0..3 ou * pour tous).

Moteurs

  • Visuel : robot vue de dessus avec SVG des roues. Les rayons tournent à un rythme ∝ vitesse. Reverse = sens inverse. Lignes de mouvement aux hautes vitesses.
  • Contrôles : vitesse −255..+255, direction, frein, vitesse de rampe.
  • Calibration : trim par roue (offset pour rouler droit), sauvegardé.
  • Verbe : M:L,R (signé −255..255 chacun).

Buzzer

  • Visuel : SVG haut-parleur avec ondes sonores concentriques (rayon = volume, teinte = fréq). Clavier piano en bas. Web Audio joue le même ton dans le navigateur.
  • Contrôles : fréquence, durée, notes presets, sélecteur de mélodie.
  • Verbe : BUZZ:f,ms.

Ultrason

  • Visuel : arc de balayage sonar depuis l'avant du robot. Point distance épinglé sur l'arc. Grille polaire (10–100 cm). Traînée des 50 dernières lectures en points qui s'effacent. Bandes de seuil vert/jaune/rouge configurables. Lecture chunky.
  • Contrôles : taux de polling, bouton ping manuel, zones de seuil.
  • Calibration : offset zéro.
  • Verbe : DIST? → DIST:cm.

Capteurs de ligne

  • Visuel : robot vue de dessous avec 2 « yeux ». En dessous : bande de ruban noir draggable que l'enfant peut faire passer sous les yeux ; les lectures réelles overlay la simulation.
  • Contrôles : taux de polling, log des transitions.
  • Verbe : LINE? → LINE:l,r.

Télécommande IR

  • Visuel : SVG photoréaliste de télécommande. La dernière touche pressée flashe + brille.
  • Contrôles : mode raw-code / mode touche nommée. Mode apprentissage : l'enfant presse une touche physique, la nomme, l'app mappe le code.
  • Verbe : événement push IR:code (ou poll IR?).

Drive Explorer (composite — utilise moteurs + télémétrie accel)

  • Visuel : joystick + bouton STOP + canvas robot fantôme (traînée de dead-reckoning).
  • Télémétrie : graphe accel 3-axes en direct avec marqueurs verticaux de commande sur la timeline.
  • Verbe : M:L,R + stream auto ACC:x,y,z ~20 Hz.

7. Surcouche de télémétrie

Dans le Drive Explorer, le panneau du bas = graphe accel 3-axes en direct avec marqueurs de commande.

Accélération (g)
  X ━━━━━╱╲━━━━━━╱╲╲━━━━     ← avant, puis tournant
  Y ━━━━━━━━━━━╱──╲━━━━━     ← latéral en virage
  Z ━━━━━━━━━━━━━━━━━━━━     ← baseline gravité
        ↑           ↑
   [▶ FWD 200]  [↻ L 150]    ← marqueurs commande sur timeline

Fonctionnalités :

  • Auto-scroll de la timeline (fenêtre 10 s, drag à deux doigts pour l'historique).
  • Hover n'importe quel point → tooltip avec valeur g + commande la plus proche.
  • Cliquer un marqueur → surligne le segment de courbe après lui.
  • Pause / play / snapshot (export PNG) / comparer-runs.

Réutilisable entre explorers via js/telemetry.js :

  • Servo : angle commandé vs temps + accel si bras en mouvement.
  • Moteurs : commande + accel + (plus tard) encodeurs roue.
  • Buzzer : forme d'onde réelle capturée par micro.
  • Ultrason : distance vs temps + marqueurs ping manuel.

8. Cadre add-ons (stubbed en v0.1.0)

Réserver l'espace de noms des verbes maintenant pour que v0.2.0+ ne casse pas le protocole :

ADDON:LIST                       → ADDON:none|<name>,<port>|...
ADDON:READ:<port>                → ADDON:<port>:<value>
I2C:SCAN                         → I2C:0x10,0x3C,0x76,...
I2C:READ:<addr>:<reg>:<n>        → I2C:<addr>:<reg>:<bytes>
I2C:WRITE:<addr>:<reg>:<bytes>   → ECHO

Le firmware livre ces verbes stubés (retour none / vide). Ajouter de vrais add-ons Gravity/I2C plus tard = drop-in de fichiers js/addons/<name>.js, pas de bump de protocole.

9. Identité visuelle

  • Thèmes : système 5 thèmes (Carbon · Forest · Steel · Paper · Pearl) — vibe labo d'inventeur amical
  • Statuts : vert/ambre/rouge avec variantes en glow subtil
  • Typographie : Righteous (display) · Tajawal (body, aussi arabe) · JetBrains Mono (code)
  • Mode : sombre d'abord (Carbon par défaut) ; 2 thèmes clairs (Paper, Pearl)
  • Animation : physique de ressort, jamais linéaire
  • Son : off par défaut, prompt « 🔊 activer »
  • Microinteractions : hover scale 1.02, active 0.98, focus ring avec glow
  • Haptique : Vibration API sur snap de slider mobile + hits de bouton

Couche polish (construire une fois, réutiliser partout)

  • SVG carte Maqueen partagé (utilisé comme base dans chaque Explorer).
  • Système de particules (~50 lignes).
  • Préset filtre CSS bloom.
  • Util physique de ressort.
  • Widget jauge configurable (arc / ticks / aiguille).
  • Kit son (synth Web Audio, pas de fichiers audio).

Discipline de stack

  • SVG + CSS uniquement — pas de moteur 3D.
  • Web Animations API native — pas de bibliothèque d'animation.
  • JS vanilla — pas de framework, conserve le feel léger de bit-playground.

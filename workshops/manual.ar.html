SOPالجدول الزمني للعملية

// المرحلة 1 · الاتصال (10 د)

  • اِجمع العملاء في نصف دائرة. اعرض الوحدة النائمة.
  • تساءَل: "كيف تصل إشارة الأمر إلى هذه الوحدة؟" اجمع الفرضيات (واي-فاي، تخاطر، سحر، أشعة كونية). اعترف بالكلّ، صادِق على البلوتوث.
  • عرض: المُدرِّب ينقر ⌘ CONNECT. مؤشر الوحدة يستجيب. تقدّم ثانية واحدة. توقّف.
  • أعلِن أداتَي التحكم: الأزرق ⌘ = اتصال، الأحمر ⏹ = إيقاف. هذا هو اليوم بأكمله.

// المرحلة 2 · تدريب المصافحة (10 د)

  • قسّم المجموعة. كلٌّ يأخذ جهازاً + وحدة.
  • انقر ⌘ ← قائمة BLE ← اختر "Maqueen…" ← أكِّد Pair.
  • زرّ FAB يختفي عند النجاح = المصافحة مؤكّدة.
  • كل مجموعة: اتصال، فصل، إعادة اتصال × 2. بناء ذاكرة عضلية.
فشل الاقتران = خطأ من جانب الجهاز، ليس الوحدة. أوقف/شغّل البلوتوث على الجهاز. أعد تشغيل التطبيق.

// المرحلة 3 · استطلاع حر (20 د)

  • "لديكم 20 دقيقة. قُد. لا أهداف. لا تكسر شيئاً."
  • المُدرِّب يبقى يداه في جيبيه. راقب. سجّل مَن سيحتاج المساعدة لاحقاً.
  • بعد 10 د: تلميح غير رسمي — "هل اختبر أحدكم منزلق السرعة؟"
  • 5 د متبقّية: "نفّذ دوران 360° مثاليّاً."

// المرحلة 4 · عقيدة زر القتل (10 د)

  • كل الوحدات متوقّفة. الأجهزة مطويّة.
  • قُد بسرعة 100% نحو قدم المُدرِّب. قبل نصف ثانية من الاصطدام: انقر ⏹. الوحدة تتجمّد.
  • المجموعة تكرّر بصوت واحد: "أحمر = إيقاف. متاح دائماً."
  • وقّعوا قانون السلوك التنفيذي أدناه.

// قانون السلوك التنفيذي للعميل

أنا، العميل __________________________، أتعهّد بـ:

  1. الحفاظ على الوعي الموقعي بزرّ ⏹ القتل الأحمر في جميع الأوقات.
  2. عدم توجيه الوحدة نحو الأشخاص أو الحيوانات أو المعدات.
  3. الإيقاف افتراضياً عند أيّ شك.

التوقيع: __________________________   التاريخ: ____________

// المرحلة 5 · تمرين التعرّج (25 د)

  • اضبط 4-6 أقماع في زيغ-زاغ، تباعد 50 سم.
  • الهدف: العبور دون احتكاك.
  • الجولة 1: ركضة واحدة لكل عميل. مُؤقَّتة. مَن يلامس قمعاً يُستبعد.
  • الجولة 2: العميل يختار السرعة (10/30/50/80%). معظمهم يفشلون عند 80% — درس طبيعي.
  • الجولة 3: عَصْب أعين القائد. الجناح يقدّم متجهات شفهية. تدريب ثقة + اتصالات.

// المرحلة 6 · الاستخلاص (15 د)

  • دائرة. الوحدات خارج الخدمة. الأجهزة مودَعَة.
  • كل عميل يُبلِّغ: 1× أصعب لحظة، 1× اكتشاف غير متوقّع.
  • المُدرِّب: "أنتم الآن عملاء تنفيذيّون. العملية القادمة: طيّارون."

// المرحلة 1 · إحماء (5 د)

  • سباق المصافحة السريع. أوّل المتّصلين يفوز.
  • سؤال السلامة: "أين ⏹؟" — كل العملاء يشيرون.

// المرحلة 2 · تجربة 3 سرعات (15 د)

  • ضع الدواسة عند 10%. تقدّم 3 ثوانٍ. علِّم موضع التوقّف.
  • أعد الضبط. ضع عند 50%. تقدّم 3 ثوانٍ. علِّم.
  • أعد. ضع عند 100%. تقدّم 3 ثوانٍ. علِّم. قارن المسافات. نفس الزمن، إزاحات مختلفة جدّاً.

// المرحلة 3 · تصوير المكابح (20 د)

  • اِشريط خطّ البداية على الأرض. الوحدة خلفه.
  • تقدّم. اضغط ⏹ عند عبور المقدّمة لخط البداية.
  • علِّم موقع التوقف الفعلي. القياس = مسافة الكبح.
  • أعد عند 30%، 60%، 100%. 3 محاولات لكلٍّ. جدوِل.

// المرحلة 4 · رسوّ دقيق (25 د)

  • اِشريط مربّعاً 30×30 سم على بعد 1 متر.

  • 3 محاولات. العميل يختار الدواسة.
  • التقييم: 3 = داخلياً تماماً. 1 = على الخط. 0 = خارج.

// المرحلة 5 · تخطيط البيانات (15 د)

  • كل عميل: ورقة بيانية.
  • المحور س: الدواسة %. المحور ص: مسافة الكبح (سم).
  • اِخطِّط 3 نقاط. اِربطها بخط.
  • السؤال: "هل الخط مستقيم؟" (لا) المنحنى يرتفع.

// المرحلة 1 · تدريب الإعماء (10 د)

  • كل عميل بعصابة. الجناح يُعطي متّجهاً شفهيّاً فقط.

// المرحلة 2 · القناة A · سونار (20 د)

حقيقة: الخفافيش تُحدّد الفرائس في الظلام الدامس باستخدام تحديد المسافة فوق الصوتي. نفس الفيزياء. SR04 يبثّ نبضة 40 كيلوهرتز ويقيس الصدى.
  • افتح شريط الحساسات الحيّ. أبرز قراءة السونار.
  • يد على بعد 10 سم ← اقرأ القيمة. 50 سم ← اقرأ مجدّداً.

// المرحلة 3 · القناة B · حساسات الخط (15 د)

  • ضع شريطاً أسوداً على أرض بيضاء (50 سم).
  • حرِّك الوحدة يدويّاً عبره. راقب قيم L/M/R.
  • أسود = عائد منخفض. أبيض = عائد مرتفع. انعكاس بصري.

// المرحلة 4 · توثيق الشبكة (25 د)

  • وزِّع مخطط الوحدة فارغاً.
  • سَمِّ كل حساس: اسم · قناة · وحدة · متجه (إلى أين يشير).
  • تشفير اللون: كهرماني=سونار، أخضر=خط، برتقالي=IR، سماوي=LED.
  • أفضل الخرائط على الجدار.

// المرحلة 5 · تدريب التنبّؤ (10 د)

  • المُدرِّب يُقدّم: يداً · كتاباً · وسادة · كأساً.
  • اكتب توقّع السونار قبل القياس. كَشْف. الأقرب = نقطة.
  • شذوذ: الوسادة تمتص فوق الصوتي ← قراءة أبعد من الواقع.

  • اعرض 3 خرائط: مترو · مخطط طابق · أقمار صناعية.
  • تساءَل: "ما البيانات المُشَفَّرة؟ ما المُحذوف؟"

// المرحلة 2 · بناء حيّ (20 د)

  • افتح علامة المرحلة. لوحة فارغة.
  • تقدّم 1 متر ← يظهر متّجه أخضر.
  • قُد مربعاً 1×1 متر ← يُرسَم المسار.
  • نبضات السونار = علامات حمراء.

// المرحلة 3 · تصوير الانحراف (15 د)

  • ضَع علامة بداية.
  • قُد مربعاً 2×2 متر. عُد للأصل.
  • قِس: الموضع الحقيقي مقابل الموضع المُقدَّر للخريطة.
  • الفجوة = الانحراف. قِسها بالسنتيمتر.
مصادر الانحراف: انزلاق العجلة · تفاوت السطح · معايرة محرك · ضوضاء حساس. كلها واقعية. كلها تراكمية.

// المرحلة 4 · تجربة مقارنة (15 د)

  • وحدتان · نفس الطريق. قارن الخرائط جنباً إلى جنب.
  • "متطابقتان؟" لا. كل وحدة تنحرف بشكل فريد.

// المرحلة 5 · عقيدة GPS (10 د)

  • "تخيّل سيارة ذاتية القيادة باستخدام قياس المسافة فقط."
  • احسب: 1% انحراف × 10 كم = 100 متر. فشل المهمة.
  • الحل: اندماج الحساسات = قياس المسافة + GPS + كاميرات + IMU + ليدار.

// المرحلة 6 · خريطة تكتيكية (15 د)

  • اِرسم غرفتك (أو طريق المدرسة) بأسلوب SLAM.

هندسة المحرك الدقيق

"محرّك مُتحَكَّم بالموضع. دقّة 0-180°. صَمِّم حِملك."
المدة: 90 دالصعوبة: ▰▰▰▱▱المتطلبات: OP-01

[منفذ S1/S2 نشط] السيرفو يختلف عن المحرك العاديّ. سيرفو = زاوية دقيقة. محرك = دوران مستمرّ. تأمر بـ 47°. السيرفو يذهب إلى 47° بالضبط. مُستخدَم في: طابعات ثلاثية الأبعاد، روبوتات جراحية، طائرات RC، animatronics.

// المرحلة 1 · مقدمة سيرفو (10 د)

  • مرّر SG90 حقيقي. حدِّد الوصلة، القرن، الجسم.
  • اقرأ 0° / 90° / 180°. افتح SERVO. أرسِل 0°.
  • استجابة على الشاشة + سيرفو فيزيائي.

// المرحلة 2 · تدريب رد فعل الزاوية (15 د)

  • المُدرِّب: "45!" العملاء يضغطون 45° بأقصى سرعة.
  • تتابع: 75 · 135 · 0 · 180.
  • إضافة: "حيِّني." العميل يبرمج كنس.

// المرحلة 3 · وضع الكنس (10 د)

  • فعِّل كنس مستمرّ 0↔180. عدِّل التردد.
  • "موازيات صناعية؟" ← مروحة · ماسح سونار · رشّاش · مساحة سيارة.

// المرحلة 4 · تصميم الحِمل (30 د)

  • أنت المهندس.
  • صَمِّم مُلحَقاً ورقيّاً للقرن. يجب أن يستخدم 0-180°. يجب أن يخدم وظيفة تكتيكية.
  • إلهام: ناشر علم · مؤشر · برج سونار · مخلب · عصا طبل.

// المرحلة 5 · موجز التصميم (15 د)

  • 60 ثانية لكل عميل. تصويت: فائدة تكتيكية · إبداع هندسي · وضع فشل مُحتمَل.
  • أفضل تصميم = مهندس ميكانيكي.

مكتشف المسار

"خوارزمية A*. نفس منطق Google Maps. مقياس أصغر."
المدة: 90 دالصعوبة: ▰▰▰▰▱المتطلبات: OP-05, OP-06
▣Maqueen
[المُخطِّط مُشَغَّل] الخوارزميّات وصفات للحاسوب. اليوم: حلّ متاهة على ورقة. ثم شاهد خوارزمية A* تحلّ نفس الخريطة ذاتيّاً. نفس الخوارزميّة تعمل في: Google Maps · شطرنج · ألعاب فيديو · توجيه مركبات ذاتية القيادة.

SOPالجدول الزمني

// المرحلة 1 · حلّ متاهة ورقية (15 د)

  • وزِّع متاهة 5×5 مطبوعة. البداية S ← الهدف G.
  • حُلَّها بقلم رصاص. قِس الزمن.

// المرحلة 2 · موجز الإستراتيجية (10 د)

  • كل عميل يصرّح بإستراتيجيّته في جملة واحدة.
  • نتيجة: كلّها خوارزميّات. لقد اخترعتَ البحث الإرشادي للتوّ.

// المرحلة 3 · تنفيذ A* (20 د)

  • افتح مُصَوِّر الخوارزميّة. انقر A*. اختر هدفاً.
  • راقب: سماوي = حدود · كهرماني = مُستكشَف · أخضر = المسار الأمثل.
  • "هل استكشف A* كل خلية؟" لا. الخلايا الواعدة فقط.

// المرحلة 4 · تحليل مقارن (15 د)

  • جنباً إلى جنب: مسار القلم vs مسار A* الأخضر.
  • "ماذا يفعل A* أفضل؟" · يحسب كل خطوة · يُقَيِّم كل الخيارات · لا يستسلم أبداً.

// المرحلة 5 · متاهة فيزيائية (15 د)

  • اِشريط عوائق على الأرض: متاهة فيزيائية. قُد الوحدة لرسم الخريطة (SLAM).
  • A* ينفّذ ← الوحدة تتنقّل ذاتيّاً.

// المرحلة 6 · تطبيقات صناعية (15 د)

  • Google Maps "أسرع طريق" = A* في الإنتاج.
  • Spotify · شطرنج · غسّالة · إكمال تلقائي.
  • "استخدمتَ خوارزميّات اليوم. دون كتابة كود."
